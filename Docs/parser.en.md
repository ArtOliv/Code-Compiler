# Syntactic Analyzer (Parser)

**Languages:** [Portuguese](parser.md) | [English](parser.en.md)

---

## Overview

The syntactic analyzer (or parser) is responsible for receiving the sequence of tokens generated by the Scanner and verifying if they form valid structures according to the rules of the C language. If the code is correct, it builds and returns an **Abstract Syntax Tree (AST)**.

Example:

### Input (Scanner Tokens)

```text
INT            "int"
IDENTIFIER     "x"
ASSIGN         "="
LITER_INT      "10"
SEMICOLON      ";"
```

### Output (AST)

```text
Declaration
    Type (int)
    Variable (x)
        Assignment (=)
            Identifier (x)
            Literal (10)
```

---

## How it Works

The parser was built using a Top-Down Recursive Descent Parser architecture. It starts from the main rule of the program and descends through the sub-rules of the grammar.

To iterate through the tokens, the parser uses the following control methods:

* `peek()` -> Looks at the current token without consuming it.

* `advance()` -> Consumes the current token and moves to the next one.

* `check(type)` -> Returns true if the current token matches the expected type.

* `match(type)` -> Checks the type and, if it is the correct one, advances automatically.

* `consume(type, message)` -> Strictly requires a specific token type. If not found, it throws a syntactic error.

Mathematical operator precedence is guaranteed by nesting the calls within the grammar itself (e.g., Multiplication operations in term() are grouped before additions in arithmeticExpression()).

### Error Handling

To prevent the compiler from halting execution at the first encountered error, the parser implements the Panic Mode Error Recovery technique.

* When a syntactic error is found, it is recorded in the error list.

* The parser triggers the `synchronize()` function.

* This function blindly discards tokens until it finds a safe point to resume parsing (such as a `;` closing the broken command or a `}` closing the block).

* This allows the compiler to find and report multiple syntax errors in a single run.

Example of error output:

```text
Syntactic errors found:

- Expected ')' at line 1, column 10
- Expected ';' at line 4, column 5
- Expected '(' at line 8, column 8
```

## Context-Free Grammar (CFG)

The syntax of the language recognized by this compiler follows the grammar below:

```text
Program              -> int main ( ) Block

Block                -> { CommandList }

CommandList          -> Command CommandList | ε

Command              -> Declaration ;
                        | Assignment ;
                        | IfStatement
                        | WhileStatement
                        | ForStatement
                        | ReturnStatement

Declaration          -> Type Variable ( , Variable )*

Variable             -> IDENTIFIER ( = RelationalExpression )?

Type                 -> int | float | char

Assignment           -> IDENTIFIER = RelationalExpression

IfStatement          -> if ( RelationalExpression ) Block ( else Block )?

WhileStatement       -> while ( RelationalExpression ) Block

ForStatement         -> for ( ForInit ; ForCond ; ForIter ) Block

ForInit              -> Declaration | Assignment | ε
ForCond              -> RelationalExpression | ε
ForIter              -> Assignment | ε

ReturnStatement      -> return RelationalExpression ;

RelationalExpression -> ArithmeticExpression ( RelationalOperator ArithmeticExpression )?

RelationalOperator   -> == | != | < | > | <= | >=

ArithmeticExpression -> Term ( (+ | -) Term )*

Term                 -> Factor ( (* | /) Factor )*

Factor               -> ( RelationalExpression )
                        | IDENTIFIER
                        | LITER_INT
                        | LITER_FLOAT
                        | LITER_CHAR
                        | LITER_STRING
```

## How to Test

Clone the repository at this point:

```bash
git clone --branch Parser [https://github.com/ArtOliv/Code-Compiler.git](https://github.com/ArtOliv/Code-Compiler.git)
cd Code-Compiler
```

Change the `program.c` file as you wish and run:

```bash
node index.js program.c
```
 
The terminal output will first display the list of Tokens generated by the Scanner, followed by the printed Syntax Tree (or the recovered syntactic errors, if any).
